# How-to guides

Task-oriented recipes for ai-doc-organizer — one page per job. A how-to guide
assumes the reader already knows the basics (that's what
[tutorials](../tutorials/index.md) are for) and gets a specific task done.

  line.
<!-- --- SURFACE_CLI-END --- -->
<!-- --- SURFACE_WEB_UI-START --- -->
- [Use the web UI](use-the-web-ui.md) — work with ai-doc-organizer in the
  browser.
<!-- --- SURFACE_WEB_UI-END --- -->

Add a new guide as its own page ("How do I …?" — one question, one page) and
link it here **and** in the `nav:` of `mkdocs.yml` — the strict docs build
fails on any page missing from the nav.
