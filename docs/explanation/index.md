understanding-oriented pages that explain *why* things are the way they are.
No steps to follow, nothing to look up: those belong in
[How-to guides](../how-to/index.md) and [Reference](../reference/index.md).

Nothing here yet — add a page per concept (e.g. `motivation.md`,
`architecture-decisions.md`) as the project grows, and link each one here
**and** in the `nav:` of `mkdocs.yml` — the strict docs build fails on any
page missing from the nav.
